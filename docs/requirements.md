# 생산성 Todo 웹앱 MVP - 요구사항 및 UX 시나리오

## 1. 사용자 시나리오 (User Scenarios)
1. **할 일 추가**: 사용자는 입력창에 제목을 입력하고 엔터를 치거나 추가 버튼을 눌러 새로운 할 일을 생성한다. (빈 제목 방지)
2. **할 일 수정/삭제**: 생성된 할 일의 텍스트를 클릭하여 편집 모드로 진입해 내용을 수정하거나, 삭제 버튼을 눌러 목록에서 완전히 제거한다.
3. **완료 상태 토글**: 할 일 좌측의 체크박스를 클릭하여 '진행 중'과 '완료' 상태를 오간다. 완료된 항목은 취소선 등으로 시각적 구분을 제공한다.
4. **리스트 필터링 및 검색**: 상단 필터바를 통해 우선순위별, 카테고리별로 할 일을 모아보거나, 제목/태그 기반 간단한 검색을 수행한다.
5. **드래그 앤 드롭 정렬**: 각 할 일 항목의 핸들을 드래그하여 원하는 순서대로 자유롭게 배치한다. (키보드로도 접근 가능)
6. **다크모드 전환**: 시스템 설정에 맞게 기본 테마가 적용되며, 사용자가 원할 경우 상단 토글 버튼을 통해 수동으로 라이트/다크 모드를 전환할 수 있다.

## 2. 데이터 규칙 (Data Rules)
- **우선순위 (Priority)**: `High` (높음), `Medium` (중간), `Low` (낮음)의 3단계로 관리된다.
- **카테고리 (Category)**: 기본값으로 `Work` (업무), `Personal` (개인), `Study` (공부)를 제공하며, 각 카테고리에 맞는 이모지를 활용하여 시각적으로 식별한다.
- **태그 (Tags)**: 쉼표(,) 등으로 구분하여 자유롭게 문자열 형태로 입력 및 저장할 수 있다.
- **Todo 모델 핵심 필드**:
  - `id`: 고유 식별자 (UUID 또는 timestamp 기반)
  - `title`: 할 일 내용 (string, 필수)
  - `completed`: 완료 여부 (boolean, 기본은 false)
  - `priority`: 우선순위 (enum)
  - `category`: 소속 카테고리 (enum)
  - `tags`: 사용자 지정 태그 (string array)
  - `order`: 정렬을 위한 순서 값 (number)
  - `createdAt`, `updatedAt`: 생성 및 수정 시간 (timestamp)

## 3. MVP 포함 및 제외 범위 (Scope)
- **포함 범위 (In-Scope)**:
  - 브라우저 단일 사용자용 핵심 Todo 기능 (CRUD 로직)
  - 상태에 따른 즉각적인 UI 반영
  - localStorage를 활용한 데이터 영속화 (새로고침 시 상태 및 정렬 순서 유지)
  - 드래그 앤 드롭을 이용한 항목 수동 정렬
  - 시스템 감지 및 수동 조작이 가능한 다크모드 (깜빡임 방지 적용)
  - 모바일 우선(Mobile-first) 반응형 웹 UI 적용
- **제외 범위 (Out-of-Scope)**:
  - 사용자 로그인 / 회원가입 기능
  - 다른 사용자와의 협업 또는 공유 시스템
  - 백엔드 API 연동 및 클라우드 DB 등 서버 동기화
  - 푸시 알림 및 외부 캘린더 연동 로직
